The Ruby plugin in _plugins/people_generator.rb auto-generates one page per entry in _data/people.yml, and passes two variables to this layout: page.person — the full record from people.yml, plus derived fields (display_name, initials, photo URL, etc.) page.memberships — an index of every class/club/sport they appear in So profile pages always stay in sync with the rosters — you never have to manually edit /people/*. Just edit _data/people.yml or the rosters. To customize: • Change the "Appears in" heading → edit `labels.appears_in` in _config.yml. • Show additional fields (e.g. "favorite food") → add the field in _data/people.yml, then reference it here as {{ p.favorite_food }}. ============================================================================= {%- endcomment -%} {%- assign p = page.person -%}
